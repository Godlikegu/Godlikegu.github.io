数据处理与清洗部分

数据介绍

注明：本人使用的数据是.csv文件
客观数据包含：被测试人的身高，体重，年龄，性别
主要数据采集可从两方面进行入手，一方面是传感器的种类(Accelerometer,Gyroscope,Magnetometer)，另一方面是传感器佩戴的位置(RightPocket,Waist,Wrist,Ankle,Chest)

数据处理方法:

将训练集二八划分，八成为训练集，两成为验证集

1.首先是data_disposal部分，将.csv文件里面的无关数据行剔除，只保留了图片中最下方以’;’隔开的数据，代码如下

import os

folder = './UMAFall_Dataset/'   # my folder's path
dirs = os.listdir(folder)

for file in dirs:
    print(file)                              # for check

for file in dirs:
    filepath = folder + file                 #each file's path
    with open(filepath, "r") as f:
        lines = f.readlines()   # read row by row, which is called line here
    with open(filepath,"w") as f_w:
        for line in lines:
            if "%" in line :    # if %, get rid of the uesless line
                continue
            f_w.write(line)

使用data_disposal对每一个Dataset中的数据进行预处理，直接运行即可



2.使用get_feature获取我需要的特征数据以便后续训练

这里本人使用的思想是将sensor position的NO转化成不同的特征，避免重复传入sensor position,也避免不同的编号可能有大小之间的差异,导致训练器对不同身体部位的sensor有偏好

from numpy import genfromtxt


INF = 1000000000
start = 35

def Acc_data_extract(file_name):
    # there is 5 position mentioned above
    max_x = [-INF,-INF,-INF,-INF,-INF]
    max_y = [-INF,-INF,-INF,-INF,-INF]
    max_z = [-INF,-INF,-INF,-INF,-INF]
    # from the file's name to ensure the label,fall is 1
    if (file_name.find("Fall",start,len(file_name))!=-1):
        label = 1
    else:
        label = 0
        # delimeter is ';'
    my_data = genfromtxt(file_name, delimiter=';') 
    for row in my_data:
        # row[5] is the type of sensor, 0 for Accelerator
        if (row[5]==0):                    
             # row[6] is the position of sensor
            index = int(row[6])       
        # get the max value of the acceleration in x,y,z from various position(index) 
            if (abs(row[2])>max_x[index]):
                max_x[index] = abs(row[2])
            if (abs(row[3])>max_y[index]):
                max_y[index] = abs(row[3])
            if (abs(row[4])>max_z[index]):
                max_z[index] = abs(row[4])
    #x,y,z is list, label is int
    return max_x,max_y,max_z,label

3.feature_extract的介绍

import os
import get_feature
import numpy as np

 # folder's path is parameter, so that we can deal with dataset and judgeset simultaneously 
def feature(folder):               
    dirs = os.listdir(folder)
    # initial,using the first data
    res = [0.8768832683563232, 1.059326171875, 1.0078125, 4.048095703125, 1.004150390625,
           0.9856240749359131,0.375, 0.267333984375, 2.31396484375, 0.1005859375,
           0.800494372844696, 1.033447265625, 0.7763671875, 8.0, 0.210205078125]
    res_feature = [res]
    res_label = [0]

    for file in dirs:
        #each file's path
        filepath = folder + file
        #x finally become a big list consisting of 15 elements(each 5 position in x,y,z)                 
        x,y,z,label = get_feature.Acc_data_extract(filepath)
        x.extend(y)
        x.extend(z)
        # res_label is a list of label like [1,0,0,1...]
        res_label.append(label)
        #final is a ndarray which means [[],[],[]]
        final = [x]
        #absorb the new ndarray(matrix) of 15*1 by row 
        res_feature = np.append(res_feature,values=final,axis=0)
    # return two ndarray that can be trained by svm
    # actually there is no need to use np.array here
    return np.array(res_feature) , np.array(res_label)









svm分类器训练和参数调节

之前对数据进行了一系列整理之后我们才可以用sklearn提供的库函数进行训练,对应的train模块如下

import get_feature
from sklearn import svm
import feature_extract
import os
import pickle


X,y = feature_extract.feature('./UMAFall_Dataset/')
C = 1.0 
#kernel,C and max_iter have all been tested and it's the  possible best result
svc = svm.SVC(kernel='rbf', C=1.25, gamma='auto',max_iter=53).fit(X, y)


# judge_set test to ensure the hit rate
sum = 0
hit = 0

# open the files in judge set
judgeset_path = './UMAFall_Judgeset/'
dirs = os.listdir(judgeset_path)

#count the accuracy
for file in dirs:
    sum += 1
    filepath = judgeset_path + file  # each file's path
    a,b,c,d = get_feature.Acc_data_extract(filepath)
    a.extend(b)
    a.extend(c)
    res = svc.predict([a])
    if (res[0]==d): hit +=1
    # else: print(file)

hit_rate = hit/sum

print(hit_rate)


# store the model
s=pickle.dumps(svc)
# wb+ is necessary,svm.model is its name
f=open('svm.model', "wb+")
f.write(s)
f.close()
print ("Done\n")

这样一来，我们就不需要每次都重复训练，而是将我们得到的模型保存起来，成为一个.model文件，相应地，每次调用这个model我们也要有相应的读取功能model_open，如下

import pickle

import os

缺陷和改进方向

缺陷

1.数据没有被完全使用，由于给训练提供的参数数量越大，训练时间越长，且尚不清楚最终成果可以采集到的数据有哪些，本人目前只使用了Accelerator在x,y,z不同方向上的数据，另外两个sensor的数据目前没有使用，对应get_feature.py中row[5]的不同值。

2.没有利用统计规律对数据处理，如筛出不合理数据，对数据进行标准化等等，可能会受到单位的影响，如m/s和cm/s

3.训练参数缺乏合理性，由于本人编程技术和数理基础水平的限制和简化问题的需要，目前只取用了accelerator的x,y,z方向上的最大值作为是否摔倒的阈值，对于是否需要再取用身高体重等参数尚不确定，是否是越多越好呢？

4.代码可扩展性和鲁棒性较差，目前仅仅是为了训练模型而写出来

5.时间参数没有纳入考虑范围，可能会丢失部分有效信息


改进方向

1.无脑增加参数，将其他两个sensor也纳入考虑范围

2.初步处理数据时取平均值和方差，用(x-x平均值)/根号下x的方差

3.从实际情况出发，明确前端，数据库和训练模型之间的接口，对可用参数进行搜集，重新训练模型

项目目前存在的的问题

1.如何将目前得到的model如何嵌入并运用到微信小程序上去

2.实际收集的时候有哪些数据是可以被搜集的目前尚不确定，能否采集到智能设备中Accelerometer,Gyroscope,Magnetometer三个sensor的数据？如果可以采集，单位是否一致？如果不能采集，又将怎么处理？怎么重新训练模型？

3.如何应用数据库存储，在实践上实现对应的功能

4.准确率是否存在不足，还有一些功能如判断周围设备是否有同样的行为尚未完善，报警机制单一，无反馈，容易出现漏报和误报的情况。

5.数据库的隐私问题，立项书中介绍使用某差分方法进行信息保护，尚未解决

6.三个分工之间进度差距大，接口尚未明确，尚不清楚如何做出可在实际生活中运行的实物，没有明确的方向，需要进一步制定计划
